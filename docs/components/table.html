Table

Data-driven header/body table — sortable columns, row selection, row expansion (including nested tables), and multiple cell types (text/link/icon & text/badge/action icons/action menu/custom). Layout is CSS Grid, not nested flexbox rows, so column widths stay identical between the header and every body row.

Playground

Sortable (click Name or Region), selectable (checkboxes, indeterminate select-all), expandable (caret in the first column). Zebra striping is intentionally off here — it's a UX restriction on expandable tables, Table forces it off whenever both are set.

Cell types

One example of every cell type Figma's breakdown demonstrates. "Checkbox with icon & text" and "Radio with icon & text" compose the real Checkbox/RadioButton via cellType="custom" rather than a dedicated cell type each. "Select [not coded]" and "Switcher [not coded]" are deliberately not shown — no Select or Switch component exists in this library yet, and this repo doesn't fabricate a placeholder for a component that isn't real. Figma's own "[Don't use] Checkbox & text" variant is also deliberately excluded. The first column ("Text only") doubles as the "Expandable text" demo via the table's own expandable prop.

Header variations

Filled (default, grey) vs. outlined (white) header, labeled vs. icon-only columns, and the select-all checkbox. Icon-only columns get a fixed 36px width, matching Figma's isolated "Icon only" cell exactly — not a content-hug approximation.

Nested table

An expanded row rendering a second, independent Table — nesting is just ordinary composition, there's no special-cased "nested" mode. Per design decision, a nested table always keeps the default filled (grey) header and passes bordered={false}, since it sits inside the outer expanded row's own border and shouldn't paint a second outline.

Compact, zebra, outlined

Outlined header, zebra-striped rows, bare action-icon buttons (Delete disabled while the instance is running). The "Actions" column carries a real text label but still hugs its content instead of stretching — actionIcons/actionMenu columns never compete for row width, labeled or not.
