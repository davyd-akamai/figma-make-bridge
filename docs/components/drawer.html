Drawer

Overlay panel sliding in from the right — default/small/flexible sizes, an optional leading header icon, and an optional sticky footer. Focus-trapped, closes on the close button, backdrop click, or Esc.

Playground

Each size below demos a different footer composition on purpose — the footer is a plain optional slot, not a fixed 3-button schema. Which buttons appear (and whether a Link button is even present) is a per-drawer design decision, not something the component enforces.

Overflowing content — footer placement

When content is taller than the drawer, action buttons can either be passed via footer (pinned outside the scroll area, always visible) or rendered as the last elements of children (scrolls away with the rest of the content). Both are valid, supported patterns — footer doesn't force either one.
