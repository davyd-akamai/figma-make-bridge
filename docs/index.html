figma-make-bridge

A live preview of every component and page template currently migrated into this library — the single source of truth for Figma Make prototypes, extracted from Akamai's internal ADS Figma library. Each page below renders the real markup, CSS custom properties, and interaction behavior the React source produces, hand-mirrored in plain HTML/CSS/JS (no React, no build step) so it's viewable anywhere.

See the repo for source, and GUIDELINES.md for the decision log behind each component.

Components

The individual building blocks — shell chrome (header/nav/footer) and content-tier controls (button, form fields).

GlobalHeader

Top app header — logo, search, notifications, account/sign-in, responsive across mobile/tablet/desktop.

SideNavigation

Collapsible left nav rail — hover-to-expand/pin-to-lock, accordion sections, page badges.

GlobalFooter

App footer — links, copyright, responsive stacked/row layout.

Badge

General-purpose status/label badge — subtle/accent types across 7 colors, optional leading icon.

SystemBadge

Fixed NEW/BETA label badge.

Button

Primary/secondary/link/danger variants, loading state, start/end icons.

Container

Generic bordered content surface for grouping page content.

TextField

Text input with label, helper/error text, clear button, info icon.

Checkbox

Checkbox with indeterminate state, small/large sizes.

RadioButton

Radio input, small/large sizes — native name-based grouping.

TabsHorizontal

Horizontal tab strip with icon/badge/disabled tabs and a measured collapsible-overflow "..." menu.

Icons

Every icon used across components, pulled as exact vectors from Figma.

Templates

Page-shell templates composing multiple components into a full layout.

DefaultCmPageTemplate

Composes GlobalHeader + SideNavigation + GlobalFooter with a content slot — the base shell for Figma Make prototypes.
